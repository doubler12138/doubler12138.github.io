SQL学习笔记(二)

2018-04-23

继续写SQL注入的学习笔记，感觉之前第一部分写的太基础了，接下来就不写这些了，写些重要的

  • 1.第二部分主要讲一下注入的一般步骤、主要的过滤方式和代码实现。
  • 2.第三部分讲一下针对主要过滤方式的绕过方式，这些主要是针对有回显的注入。
  • 3.第四部分讲一下盲注的方式
  • 4.第五部分就写一下一些CTF注入题的思路
  • SQL注入的内容比较多，可能花的时间会比较多

Order by语句：

  • 在找到注入点后，我们一般会用order by子句判断几个字段。网上很多博客对order by语句具体查的是查询的字段数还是数据库表中的字段数讲的很模糊。我实测了一下order by猜的是查询出的列数，而不是表实际的列数。order by本来就是指定的结果如何order，针对的就是查询结果而不是原表。所以order by猜解得到的列数还跟在后端的逻辑有关:

  • 看实测，order by 2没有问题可以正常查询，但order by 3就没有查询过结果了。

  • 那么数据库的表中真的只要两个字段吗？其实不然，我们可以看到表中其实要八个字段

  • 看源码我们就能知道为什么order by 2能成功而order by 3就失败了，因为它的查询语句里面就查询了first_name和last_name两个字段。

  • 如果我们更改一下源码，改为select from users where user_id=’$id’，看看能order by 几,可以看到order by 8是能成功的，因为select 就是查询全表，而表中有八个字段。而之所以first_name变成了1，是因为我们对所有字段进行了排序user_id排在前，所以先显示出来了

判断显示字段是哪些：
  • 上一步我们通过order by子句得出了原有查询语句查询了几个字段，现在我们就需要构造payload去判断哪些查询字段是被显示的，显示在哪些位置。payload:-1’ union select 1,2,3….,n #；n就是通过order by子句得出的查询字段个数。我们通过联合查询 union select（UNION 用于合并两个或多个 SELECT 语句的结果集，并消去表中任何重复行。
    UNION 内部的 SELECT 语句必须拥有相同数量的列，列也必须拥有相似的数据类型。同时，每条 SELECT 语句中的列的顺序必须相同.）去判断，当然首先我们要让原有的查询没有查询结果，才能显示我们的查询。对于数字型注入可以将id之类的值赋为-1，因为-1这个id一般是不存在的，所以不会有查询结果，或者后面跟一个逻辑判断 and 1=2，显然是不成立的所以原有查询语句不会执行，也就没有查询结果。

可以看到我们注入出来的1，2位置都会被显示出来，那我们接下来将所有的注入结果都替换这里的1和2，这样注入结果就都会被显示出来。

开始注入：

  • 爆出当前库和数据库版本：

  • payload:-1’ union select database(),version() # database()和version()

  • 是mysql中的两个函数，用来显示当前正在链接的数据库和mysql的版本。

  • 爆出其它库：

  • payload:-1’ union select 1,schema_name from information_schema.schemata limit 0,1 #

  • information_schema是mysql数据库自带的一个库，用来储存数据库中所有库、表和字段的信息。shemata是库中的一个表，存放数据库中所有库名，schem_name就是表中存放库名的字段名。limit 0,1用来列举查询结果，limit m,n表示从第m+1条记录开始取n条结果。

  • 由于输出位在源码中限制了只输出一个字符串，所以即使我们取三条结果它也只输出了一条，这时可以使用mysql中的group_concat()函数，我理解的他的作用是将查询内容按顺序连接成一个字符串，并删除重复内容。这样我们就能显示全部查询结果了。

  • 爆对应库的数据表：

  • payload：-1’ union select 1,group_concat(table_name) from information_schema.tables where table_schema=’dvwa’ #

  • 爆对应库和表的各个字段：

  • payload:-1’ union select 1,group_concat(column_name) from information_schema.columns where table_schema=’dvwa’ and table_name=’users’ #

  • 爆数据：

  • payload:-1’ union select 1,group_concat(字段名) from 库名.表名 #

以上就是SQL注入的常规步骤和最简单的payload，具体的payload还要视源码过滤而定，构造能绕过过滤的payload的是SQL注入最终要的部分。下面就讲讲常见的过滤方式

闭合方式的过滤：
  • 在数据闭合时可不能不再时简单的单引号了，可能是双引号，括号，或者两者三者的结合，这种过滤比较简单。

过滤空格等一些符号：
  • php中有很多函数可以对字符串进行操作，例如preg_replace(),str_replace(),strtr()等，我个人认为preg_replace()被使用的多一点。他的语法是preg_repalce($pattern1，$pattern2,$str),搜索目标$str，把字符串中的$pattern1换位$pattern2

  • 可以看到我们对post的id的值做了个处理后才赋值给id变量，具体的处理是preg_replace(‘/\s+/‘,’’,$_POST[$id]),/\s+/是正则表达式，\s表示匹配任何空包字符，包括空格、制表符、换页符等等，+表示匹配子表达式一次或多次，就是匹配一个或连续多个符号。那么实际效果就是，我们输入的普通注入语句中的空格全没了。这种方式可以过滤大多数符号，具体看正则表达式怎么写（正则表达式我后面系统学的时候再细写笔记）。

关键字过滤：
  • 在注入时，我们避免不了使用mysql中的一些关键字，例如联合查询的union，所以我们可以对输入的数据进行关键字顾虑

  • 可以看到我们先定义了keyword，内容是一些用|分隔的mysql关键字，explode函数的作用就是将字符串按照特定的分隔符号分成一个数组，str_ireplace函数的作用与preg_replace类似，但ireplace表示它的比较是不区分大小写的，最终普通的payload被过滤阉割了。例如我们输入这样的payload：-1’ union select 1,group_concat(table_name) from information_schema.tables where table_schema=’dvwa’ #实际效果：

  • 常见的过滤方式大概就这三种，具体过滤因具体的正则表达和要过滤的关键字不同而不同，下一部分将讲一下常见的绕过方式。具体不同的CTF题的过滤方式千奇百怪，但万变不离其宗，知道它的如何过滤我们就能绕过。
