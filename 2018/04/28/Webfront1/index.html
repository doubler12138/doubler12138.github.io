Web前端黑客技术揭秘第1章

2018-04-28

第1章 Web安全的关键点

  • 记录今天看的第一章的重点知识，这几个关键点对理解整个Web安全，，甚至整个安全体系都都有很大的帮助。

1.1 数据与指令：

  • 数据：服务端存储（数据库，内存，文件系统等）、客户端存储（本地Cookies、Flash Cookies等）、传输中（JSON数据、XML数据等）、文本数据（HTML、JavaScript、CSS等)、多媒体数据（Flash、MP3等）、图片数据。

  • 跨站攻击发生在浏览器客户端，而SQL注入攻击由于针对的对象是数据库，一般情况下，数据库都在服务端，所以SQL注入是发生在服务端的攻击。但HTML5提供了一个新的客户端存储机制：在浏览器端，使用SQLite数据库保存客户端数据，该机制允许使用Javascript脚本操作SQL语句，从而与本地数据库进行交互。

1.2 浏览器的同源策略：

  • 计算机的本地与web是不同的层面，Web（通常成为Internet域）运行在浏览器上面，而被限制了直接进行本地数据（通常称为本地域）的读写。

  • 同源策略是众多安全策略的一个，是web层面上的策略。

  • 同源策略规定：不同域的客户端脚本在没明确授权的情况下，不能直接读写对方的资源。

  • 同域要求两个站点同协议、同域名、同端口。以 http://www.foo.com 为例，下列各站点是否同域：








站点是否同域原因
https://www.foo.com不同域协议不同，https与http是不同的协议
http://xyz.foo.com不同域域名不同，xyz子域与www子域不同
http://foo.com不同域域名不同，顶级域与www子域不是一个概念
http://www.foo.com:8080不同域端口不同，8080与默认的80端口不同
http://www.foo.com/a/同域满足同协议、同域名、同端口，只是这里多了一个目录而已

  • 客户端脚本主要指Javascript（各个浏览器原生态支持的脚本语言）、ActionScript（Flash的脚本语言），一级Javascript和ActionScript都遵循的ECMAScript脚本标准。

  • 授权往往想到是服务端对客户端访问的授权。客户端也存在授权现象。例如：HTML5新标准中关于AJAX跨域访问的情况，默认情况下是不允许跨域访问的，只有目标站点 http://www.foo.com 明确返回HTTP响应头：Access-Control-Allow-Origin: http://www.evil.com 那么 www.evil.com 站点上的客户端脚本就有全通过AJAX技术对 www.foo.com 上面的数据进行读写操作。

  • AJAX是 Asynchronous JavaScript And XML 的缩写，让数据在后台进行异步传输，常见的使用场景有：对网页的局部数据进行更新时，不需要刷新整个网页，以节省带宽资源。

  • 资源是一个很广泛的概念，只要是数据，都可以认为是资源。同源策略中的资源是指Web客户端的资源。一般来说，资源包括：HTTP消息头、整棵DOM树、浏览器存储（如：Cookies、Flash Cookies、localStorage等）

  • DOM全称Document Object Model,即文档对象模型，就是浏览器将HTML/XML这样的文档抽象成一个树形结构，树上的每个节点都代表HTML/XML中的标签、标签属性或标签内容等。

1.3 信任与信任关系：

  • 服务器与服务器、网站与网站、Web服务的不同子域、Web层面与浏览器第三方插件、Web层面与浏览器特殊API、浏览器特殊API与本地文件系统、嵌入的Flash与当前DMOM树、不同协议之间等等都存在信任关系。一个安全性非常好的网站可能有时候会因为建立了不可靠的信任关系而导致网站被黑。

1.4 社会工程学的作用：

  • 常用的社工辅助技巧有：Google Hack、SNS垂直搜索、各种收集的数据库集合查询等

1.5 攻防不单一：

  • 一次完整的渗透会利用到多种攻击手法。攻和防都得考虑道攻击手法组合的情况。

1.6 场景很重要：

  • 任何一次攻击都脱离不了具体场景
