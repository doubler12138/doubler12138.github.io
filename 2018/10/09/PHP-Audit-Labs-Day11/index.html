题目主要考察PHP反序列化漏洞的利用，在loadData函数中使用unserialize函数对变量$data进行反序列化。首先来看看这个变量我们是否可控，这个变量来自与__construct()构造函数传入的变量。在代码的最后，对Template类进行了实例化，并将post的data作为初始数据传入，所以说$data变量我们是可控的。

但是在反序列化之前对用一个if语句对变量内容进行了一次判断，第一部分是截取前两个字符，如果前两个字符为O:则返回空，目的是判断反序列化内容是否为对象。PHP可反序列类型有String、Integer、Boolean、Null、Array和Object。去掉Object以后，我们可以考虑采用数组中储存对象的方式进行绕过。

第二部分是一个正则匹配，匹配’O:任意十进制数:’，例如’O:12:’，匹配到后仍返回空。所以说将对象放入数组中进行序列化后能被匹配到。考虑一下如何绕过正则匹配，这里就要涉及到PHP反序列化函数unserialize的底层是如何实现的，我们先来看一下PHP反序列化处理部分的部分源码：

从这几部分代码我们可以清楚的了解PHP处理Object类型的反序列话的过程。先是对字符进行判断，如果是’O’则调用yy13函数，在yy13函数中对’O’字符的下一个字符进行判断，如果是’:’则调用yy17，不是则调用yy3，退出反序列化流程。在yy17函数中先是判断’:’字符后的第一个字符是数字，如果是数字则跳转到yy20函数开始反序列化，如果是’+’，则跳转到yy19。yy19函数对’+’后的第一个字符再进行判断，如果是数字则跳转到yy20，否则跳转到yy18结束反序列化过程。所有由上我们可以知道对于’O:8’和’O:+8’，PHP都能进行反序列化，唯一的区别就是’O:+8’在反序列化的过程中多经过了一次yy19函数的判断。但我们就可以使用’O:+8’来绕过第二部分的过滤。

现在我们可以进行反序列化了，那么接下来就是去找利用点。我们可以看到类销毁时调用的__destruct()函数中调用了createCache函数，而createCache函数的内容是先判断$file和$tpl是否为空，如果为空则分别赋值为$this->cacheFile和$this->template，这两个变量是在类一开始定义的共有变量（这里需要注意的是双问号”??”运算符是PHP7版本特有的，如果非PHP7的版本使用原有的三元运算语句）。然后在往$file变量表示的文件中写入$tpl变量的内容。这就是我们的利用点。

当IN_FLAG常量未定义的时程序会直接退出并显示’Access Denied’，不会输出flag。那么哪里定义了常量IN_FLAG呢。在index.php文件HITCON类中的login函数。我们来解读一下这个函数：

通过func_num_args()函数来获得当前方法的所有参数，通过list将这些参数赋值给$username和$password，然后进行数据库查询如果有查询结果，就定义常量IN_FLAG为TURE，并且遍历查询到的结果，调用loadData方法处理结果。我们再来看看loadData方法：

这个方法采取的方式我们之前有讲过。我们再来看看HITCON类中的其它方法。__construct方法给一些共有变量赋值。__conn方法连接数据库，并插入数据。__query方法用来执行SQL语句。__source方法高亮显示源码。__destruct方法使用call_user_func_array函数来调用函数,我们来看一下这个函数的定义

call_user_func_array** ( callable $callback , array $param_arr )

把第一个参数作为回调函数（`callback`）调用，把参数数组作（`param_arr`）为回调函数的的参数传入。

__wakeup方法是魔法方法，在对象被反序列化前会自动调用，在这个方法中对类中的args数组进行遍历，对变量值进行处理，mysql_escape_string函数会将SQL语句中的特殊字符进行转义,trim函数去掉两边空字符，strtolower函数将所以字母小写。所以说当HITCON类有两个默认参数，一个是method，决定执行哪个自定义方法，可以选择的是login方法和source方法，一个是args，为一个数组，为login函数执行SQL查询时提供参数。

而在index.php文件的最下面有这么一段代码：

当有data参数传入时对data的值进行反序列化，如果没有就执行HITCON(‘source’,array())，也就是执行HITCON类中的source方法，args数组为空。所以说我们只要构造一个HITCON(‘login’,一个数组)的序列化结果，然后用data参数传入，进行反序列化后，就会调用login方法，如果根据数组中的内容进行的SQL查询有结果，那么IN_FLAG常量就会被定义。但是除此之外我们需要将flag.php文件包含到index.php文件中，所以我们需要一个包含点，也就是soFun类。它会在__destruct方法中包含类中file变量中的文件，但是里面有一个__wakeup方法，当soFun类被反序列化时，file变量会复制为index.php。而且我们需要第二个反序列化点来反序列化soFun类，也就是HITCON类中的loadData方法，它会对login方法中SQL查询的结果是进行反序列化。所以我们需要将soFun类的序列化后的值作为SQL查询的值，也就是要进行SQL注入。

所以现在我们先来理一下目前的思路：

首先，我们需要传入一个data的值，该值是HITCON类的序列化结果，调用login方法，并将soFun的序列化结果作为args中username或password的值用SQL注入的方式注入到查询语句中。

其次我们需要绕过两个__wakeup方法，HITCON类的__wakeup方法会对我们args数组中的值做处理，对我们的注入的语句产生影响，soFun类中的__wakeup方法会直接确定包含文件的名字。

我们先来看payload，在来分析为什么是这样的payload，payload如下：

分析一下这个payload，O表示object是一个对象，6便是对象名长度为6，HITCON是对象名，3表示有对象中有三个属性，大括号内的就是属性。s表示属性名为String类型，6和method分别是属性名和长度。s表示属性值是String类型，5和login是属性值和长度。第2个属性名是args，它的类型是array类型，里面有两个键值对分别是username，他的值是1' union select 1,2,'a:1:{s:2:"xx";O:%2b5:"SoFun":2:{s:4:"file";s:8:"flag.php";}}'%23，可以看出这明显是一个注入语句，联合查询了username为1的结果和select的结果，这些结果都会被loadData方法反序列化。还有一个是password，他的值是乱写的，因为它最后会被注释掉，所以多少也就无所谓了。有两个属性，为什么一开始是我们构造的序列化的属性数设置为三，这就是我们绕过__wakeup方法的方法。这是由一个已知CVE漏洞导致的，对于PHP版本PHP5 < 5.6.25 PHP7 < 7.0.10 的版本，当被反序列化的字符串中对应的对象属性数大于真实个数时__wakeup方法不会被执行但反序列化仍然有效。

分析完这个payload，我们来分析一下这个payload是如何执行的。首先，代码会反序列化data的值，从而生成一个HITCON对象，当这个类被销毁时，同样会调用源代码中HITCON类中的__destruct方法，__destruct方法会将method和args作为参数调入call_user_func_array函数，call_user_func_array函数会调用method变量所表示的函数，也就是login，并将args作为参数传入。args作为数组有两个值会拼接到login中的查询语句，最后形成查询语句SELECT * FROM users WHERE username='1' union select 1,2,'a:1:{s:2:"xx";O:%2b5:"SoFun":2:{s:4:"file";s:8:"flag.php";}}'%23' AND password='md5(234)'

最后的查询结果，也就是users表中username为1的结果和1、2以及a:1:{s:2:"xx";O:%2b5:"SoFun":2:{s:4:"file";s:8:"flag.php";}}，因为有结果，所以IN_FLAG常量会被定义，而这些结果会传入loadData方法中进行反序列化。a:1:{s:2:"xx";O:%2b5:"SoFun":2:{s:4:"file";s:8:"flag.php";}}反序列化后是一个数组，之所以是数组是为了绕过loadData中的过滤，数组里有一个类。%2b是’+’，也是为了绕过过滤。类中的属性为2，实际上只有1个，属性名是file，属性值是flag.php，这是为了绕过soFun类中的__wakeup方法，这样包含文件名就会成为flag.php。而这个IN_FLAG常量已定义，所以就会flag就会被echo出来。最终结果如下：
