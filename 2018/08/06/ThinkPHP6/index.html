SQL查询

ThinkPHP可以通过query方法和execute方法执行原始的SQL查询。

query方法：执行读取操作，如果成功返回数据集，失败返回false。

execute方法：执行写入操作，执行成功返回受影响的行数。失败返回false。

自动验证

自动验证是ThinkPHP模型层提供的一种数据验证方法，可以在使用create创建数据对象的进行数据验证。

数据验证有两种方式：

  • 静态方法：在模型类中通过$_validate定义
  • 动态方法：使用模型类的validate方法

1.验证规则

验证规则如下：

  • 字段名：需要验证的表单字段名称，不一定是数据库字段
  • 验证规则：要进行验证的规则，需要结合附加规则。系统内置了一些常用正则验证的规则，可以直接作为验证规则使用
  • 错误提示：用于验证失败后的提示信息定义
  • 验证条件：可选（0：粗在字段就验证（默认），1：必须验证，2：值不为空的时候验证）
  • 附加规则：可选
  • 验证场景：可选（1：新增数据时验证，2：编辑数据时验证，3：全部情况下验证（默认））

自动完成

自动完成是ThinPHP提供用来完成数据自动处理和过滤的方法，使用create方法创建数据对象的时候会自动完成数据处理。一次，在ThinkPHP使用create方法来创建数据对象是更加安全的方式。自动完成通常用来完成默认字段写入、安全字段过滤以及业务逻辑的自动处理。自动完成的定义支持动态定义和静态定义两种方式：

  • 动态方式：使用模型类的auto方法动态创建自动化规则
  • 静态方法：在模型类里面通过$_auto属性定义处理规则

定义规则

两种方法的定义规则是一致的，一般格式为：

  • 完成字段：需要自动完成的字段名
  • 完成规则：以何种规则处理该字段
  • 完成条件：可选，自动完成的执行场景，场景如下
    • Model::MODEL_INSERT：新增数据的时候处理（默认）
    • Model::MOEDL_UODATE：编辑数据的时候处理
    • Model::MODEL_BOTH：所有情况都处理
  • 附加规则：可选，规则如下：
    • function：函数，函数定义在function.php中
    • callback：回调方法，方法定义在当前模型类中
    • field：使用其他字段填充
    • string：字符串填充（默认）
    • ignore：为空，则忽略
