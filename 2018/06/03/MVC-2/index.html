    .htaccess文件（分布式配置文件）提供针对目录改变配置的方法，即在一个特定的文档目录中防止一个包含一个或多个指令的文件，以作用于此目录及其所有子目录。其可以帮我们实现：网页301重定向、自定义404错误界面、改变文件扩张名、允许/阻止特定的用户或者目录的访问、禁止目录列表、配置默认文档等功能

重定向的原因：

  • 静态文件能直接访问：

    如果文件或者目录真实存在，则直接访问存在的文件/目录

    比如，静态文件static/css/main.css真实存在，就可以直接访问它

    程序有单一的入口：

    这种情况时请求地址不是真实存在的文件或目录，这样请求就会传到index.php上

    例如，访问地址：`localhost/item/view/1,在文件系统中并不存在这样的文件或目录

    那么Apache或Nginx服务器会把请求发送给index.php，并把域名之后的字符串赋值给REQUEST_URI变量，这样在PHP中用$_SERVER[‘REQUEST_URI’]就能拿到/item/view/1

    可以用来生成美化的URL，利于SEO：

    SEO优化，就是让网站在搜索引擎中排位更靠前

配置文件

创建配置文件：

在入口文件中我们加载了config.php文件，其作用是定义数据库连接参数，以及配置默认控制器名和操作名，其内容如下：

<?php

define(‘DB_NAME’，’project’);
define(‘DB_USER’, ‘root’);
define(‘DB_PASSWORD’, ‘root’);
define(‘DB_HOST’, ‘localhost’);

$config[‘defaultController’] = ‘Item’;
$config[‘defaultAction’] = ‘index’;

return $config;
​

文件分析：

定义了数据连接所需要的数据库名、用户名、密码和允许连接用户的常量，设置了一个数组，默认控制器名为Item，默认操作名为index。

return写在函数外面是PHP的特殊用法之一，在同一个文件中，如果执行了return，则return 语句后面的内容将不会被执行了；若文件调用另一个文件中的方法或其他代码，在该方法或代码中使用return 则该方法后面的语句不再执行，但调用该方法或代码的源文件照常执行。并且return的值会当作require的返回值
