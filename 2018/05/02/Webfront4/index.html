Web前端黑客技术揭秘第2章（3）

2018-05-02
  • 学习笔记

第2章 前端基础

  • 这部分主要记一下Cookie安全和本地储存风险。

2.5.4 Cookie安全：

  • 同域内浏览器发出的任何一个请求都会带上Cookie，请求是，Cookie会出现在请求头的Cookie字段中。服务端响应头的Set-Cookie字段可以添加、修改和删除Cookie，大多数情况下，客户端通过JavaScript也可以添加、修改和删除Cookie。

  • Cookie常被用来储存用户的会话信息，比如说用户登陆认证后的Session，之后同域内发送的请求都会带上认证过后的会话信息。

  • Cookie的重要字段：name value domain path expires httponly secure. 其一次含义是：名称、值、所属域名、所属相对路径、过期时间、和i否有Http Only标志、是否有Secure标志。

  • 1.子域Cookie：这是domain字段的机制，设置Cookie是，如果不指定domain的值，默认就是本域。例： a.fo.com 域通过JavaScript来设置一个Cookie：document.cookie=”text=1”; 此时，domain值默认为a.foo.com。 a.foo.com 域设置Cookie是，可以指定domain为父级域，比如：document.cookie=”test=1;domain=foo.com”; 这样做的好处是在不同的子域共享Cookie，坏处就是攻击者控制的其它子域也能读到这个Cookie。另外这个机制不允许设置Cookie的domain为下一级子域或其它外域。

  • 2.路径Cookie机制：这是path的机制，设置Cookie是，如果不指定path，默认就是目标页面的路径。通过指定path字段，JavaScript有权限设置任意Cookie到任意路径下，但是只有目标路径下的页面JavaScript才能读取到该Cookie。但通过跨iframe进行DOM操作可以跨路径读取Cookie。

  • 3.HttpOnly Cookie机制：HttpOnly是指仅在HTTP层面上传输的Cookie，但设置了HttpOnly标志后，客户端脚本就无法读写该Cookie，也要就能有效防御XSS攻击获取Cookie。如果服务端响应的页面有Cookie调试信息，就很可能会导致HttpOnly Cookie泄露：1)PHP的phpinfo()信息；2)Django应用的调试信息；3)Apache Http Server 400错误暴露HttpOnly Cookie.Apache HTTP Server 2.2x多个版本没有严格限制HTTP请求头信息，HTTP请求头信息超过LimitRequestFieldSize长度是，服务器返回400（Bad Request）错误，并在返回信息中将出错的请求头内容（包括请求头里的HttpOnly Cookie），攻击者可以利用这个缺陷获得HttpOnly Cookie.

  • 4.Secure Cookie机制：指的是设置了Secure标志的Cookie仅在HTTPS层面上安全传输。如果请求是HTTP的，就不会带上这个Cookie，但是SecureCookie对于客户端脚本来说是可读写的。

  • 5.本地Cookie与内存Cookie：与过期时间（Cookie的expires字段）紧密相关。如果没设置过期时间，就是内存Cookie，这样的Cookie会随着浏览器的关闭而从内存中消失；如果设置了过期是未来的某个时间点，那么这样的Cookie就会以文本的形式保存在操作系统本地。很多网站未来提升用户体验，不用每次登陆，采用本地Cookie的方式，但这样风险也变大了。攻击者通过XSS得到这样的本地Cookie后，就能在未来很长一段时间内，甚至永久控制着目标用户的账号权限。但内存Cookie也可以人为的添加过期时间使其编程本地Cookie。

2.5.5 本地存储风险：

  • 1.Cookie：大多数浏览器限制每个域能有50个Cookie。不同的浏览器能存储的Cookie试有差异的，其最大值约为4kb。

  • 2.userData:微软在IE5.0以后，自定义了一种持久化用户数据的概念uerData，用户数据的每个域最大为64kb。

  • 3.localStorage:HTML5的本地存储localStorage是大势所趋，如果仅存储在内存中，则是sessionStorage，它们语法都一样。基础语句：localStorage.setItem(“a”,”xxxxxxxxx”); localStorage.getItem(“a”); localStorage.removeItem(“a”); 5大浏览器都支持以localStorage方式进行存储，处理Oprea浏览器采用BASE64编码以外，其它都是以明文存储。在时效性上，localStorage并不会想Cookie那样可以设置数据存活时限，除非用户主动删除，不然永久存在。基于以上原因，并不建议使用localStorage方式存储敏感信息。

  • 4.Flash Cookie:Flash是跨浏览器的通用解决方案，Flash Cookie默认存储数据大小是100KB。
