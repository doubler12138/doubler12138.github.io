二刷南邮CTF平台的web题，写个wp，为实验班考试做准备

签到题：

  • 打开题目地址，F12出flag

md5 collision:

  • 考察php弱类型，题目界面给了源码

    分析一下就是md51是字符串md5加密以后的结果，md52是GET传参md5加密后的结果，传入的参数不能是原字符串，但加密结果要与原字符串加密结果一样。但用的是两个等号。原字符串加密后是0e开头的。

    那么我们只要传入一个字符串加密后是0e开头的,在比较这连哥哥字符串时，php会将0e这类字符串识别为科学计数法，0无论多少次都等于0，就能让两者相等。

    php弱类型总结

签到2：

  • 题目地址界面让输入登陆口令，但是口令11个字，但输入框输入长度限制为10。F12改一下前端限制。

这题不是WEB：

  • 这题是MISC图片隐写，题目地址界面把gif图片保存到本地，winhex打开看一下，出flag

层层递进：

  • 打开题目地址，什么都没发现，F12，刷新一下，在网络里看到它请求了一个特殊的文件，404.html.看一下返回信息，拼接出flag

AAencode:

  • AAencode是一种加密方法，可以把javascript加密成颜文字。这道题我看了很多wp说是直接在控制台里跑一下加密后的颜文字就行了，但我火狐和edge都没跑出来，只能找一个解密网站解密了。

单身二十年：

  • 单身100年都没用，打开题目地址，有一个链接，点一下，直接跳转，到一个没用的界面。用burp抓包一下，发现中间访问了一个页面，这个页面上有flag，但它直接跳转到那个没用的界面。所以没有burp，单身100年也没用。

你从哪里来：

  • burp抓包改一下referer头，改成http://www.google.com，按理说这样就没问题了，但是出不来flag，查了很多wp也是这样

php decode:

  • 看一下源码，一个解密函数，对参数进行base64解密，再gzinflate解密，复制到本地，或者找个在线运行，运行一下就行，。但是要把eval改成echo，才能输出结果。

文件包含：

  • 进入题目地址，有一个链接，点击，跳转到另一个网页。网页有一个file参数，读取一个show.php文件，既然是文件包含，就用php伪协议读取index.php。过滤器base64。读出来后，base64解码一下，flag写在注释里面。

单身一百年也没用：

  • 题目界面有一个链接，burp打开，点击，302重定向。先访问了/web9/index.php，重定向到一个没用的界面，flag写在/web9/index.php的返回头里。

bypass again:

  • php弱类型，看一眼给的源码，get传参a,b值不能相等，但md5加密后要相等。第一种做法，传两个数组进去，注意数组的值不能相等，或者干脆不给值。数组无法md5加密，但是因为它是==，所以还是判定其相等，出flag。第二种就是和web2一样的做法

变量覆盖：

  • 看它给的源码，只要变量pass和变量thepassword_123相等就能出flag

    register_global=ON时POST传入数据值会覆盖掉php中变量值，源码中对post数据进行了extract()处理，extract()函数会将数组中的值赋给对应索引，故传入pass=1&thepassword_123=1即可。

  • 题目给出了源码，有了查询语句，闭合一下，万能密码绕过,admin’) and 1=1 #

pass check:

  • 弱类型，传数组

起名字真：

  • 简单的代码审计，看一下它给出的源码就行了。

    GET传参key的值，ord函数返回单个字符的ASCII码值，所以我们传进去的key的值不能是数字，但要等于54975581388。传数字的十六进制就行了

密码重置：

  • 要求重置admin的密码，进入题目地址后发现账号是ctfuser，而且无法更改。

    这里要注意它的URL，上面有一个user1参数，里面的内容是ctfuser base64加密再url编码过后的内容。所以我们把参数改成admin的base64加密后的内容再post传参user=admin，密码无所谓，验证码填它的就行了

php反序列化：

  • 题目环境炸了，我用它给的源码提示做了一下本地复现，首先看它的源码

    它把我们get传参进去的内容反序列化了，并在这之前定义了一个类，类里面有两个变量，enter和secret.然后它把反序列化后的结果下的secret复制为flag，源码提示用*代替。那么我们传进的参数最起码是一个包含secret变量的类。我把这段源码抄到本地，假设我们的flag就是flag

    思路很清晰了，我们要构造一个php序列化的结果，内容是一个跟它一样的类。里面有enter和secret变量，而且要让它们的值相等。但是我们并不知道flag，所以不可能直接给enter变量赋值，使他等于在后面被赋值的secret变量。这里就要涉及到$a=$b和%a=&%b的区别了。$a = $b;是把$b的值赋值给$a,如果赋值后,$b的值改变了,$a的值是不会改变的$a =&$b;这个是把$b的地址传递给$a,这个时候,$a和$b都指向内存中的同一块地址,所以如果赋值后,你改变了$b的值,$a的值也是会改变的。所以我们只要$enter=&$secret，即使在后面，secret被重新赋值了，两者还能相等。写构造poc的程序

    poc:O%3A8%3A%22just4fun%22%3A2%3A%7Bs%3A5%3A%22enter%22%3BN%3Bs%3A6%3A%22secret%22%3BR%3A2%3B%7D

    只要我们把这个传进去就行了

sql injection4:

  • 题目提示反斜杠可以用来转义。进入题目界面，给了源码提示，复制到本地格式化一下。目标是在不知道用户民和密码的情况下登陆。

    这里它自定义了一个clean函数，对我们get传进去的username和password参数进行了过滤，函数里面用了htmlentities函数，设置了参数ENT_QUOTES,作用
    将双引号和单引号编码成html实体，所以我们没有办法传入单引号来提前闭合。但是我们可以用它查询语句里面的单引号来闭合，仔细看它的查询语句：

    select * from users where name='$username' and pass='$password';

    这样我们就可以很好的利用了，我们传入$username为反斜杠\，把后面的单引号转义了，这样第一个单引号和第三个单引号形成闭合，而第二个单引号被单纯的看作字符，然后我们传password为 or 1=1 %23，把第四个单引号注释掉。

    访问一下，提示TIP在它脑袋里，就是返回头呗

    linux服务器下的命令行历史记录/.bash_history呗，在文件路径访问一下

    有一个flagbak.zip，访问下载一下，解压出flag

SQL注入2：

  • 题目提示union查询，进入题目界面，给了源码链接，点进去看一下。

    思路就是，提前闭合查询语句，且让查询结果为空，然后union select随便哪个值的md5加密结果，让查询返回结果为union select的值就行了，然后密码输入那个值就行了。

    随便给两个payload：

    user=' union select 'c4ca4238a0b923820dcc509a6f75849b'#&pass=1

    或者

    user=' union select md5(1)#&pass=1
