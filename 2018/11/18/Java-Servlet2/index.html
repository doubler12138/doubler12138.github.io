Java-Servlet学习笔记（二）--Servlets

字数统计: 1,847字   |   阅读时长: 7分
  • Java
  • Servlet

前言

开始学习Java ee的基础Servlet，博客以笔记的形式，可能写的比较乱。

Servlet API

1.Servlet API有一下四个包：

  • javax.servlet 包含定义Servlet和Servlet容器之间契约的类和接口
  • javax.servlet.http 包含定义HTTP Servlet和Servlet容器之间契约的类和接口。
  • javax.servlet.annotation 包含标注Servlet、Filter、Listener的标注。它还为被标注原件定义数据
  • javax.servlet.descriptor 包含提供程序化登陆web应用程序的配置信息的类型

2.在一个应用程序中，每种Servlet类型只能有一个实例

3.用户请求流程：

用户请求知识Servlet容器调用Servlet的service方法，并传入一个ServletRequest实例和一个ServletResponse实例。ServletRequest实例中封装了当前的HTTP请求，ServletResponse表示当前用户的HTTP响应。

4.对于每一个应用程序，Servlet容器会创建一个ServletContext实例。这个对象中封装了上下文（应用程序）的环境详情。每个上下文只有一个ServletContext。每个Servlet实例也都有封装一个Servlet配置的ServletConfig。

Servlet

1.Servlet接口中定义了一下5个方法：

java方法签名的惯例：对于与包含该方法的类型不处于同一个包中的类型，要使用全类名。因此service方法javax.servlet.ServletException的签名中是没有包信息的，而java.io.Exception则是编写完整的名称。

2.生命周期方法：

  • init 当该Servlet第一次被请求时，Servlet容器会调用这个方法，这个方法在后续请求中不会再被调用，可以利用这个方法来执行相应的初始化工作
  • service 每当请求Servlet时，Servlet容器就会调用这个方法
  • destroy 当要销毁Servlet时，Servlet容器就会调用这个方法。一般会在这个方法中编写清除代码

3.非周期方法：

  • getServletInfo 这个方法会返回Servlet的描述。可以返回有用的或为null的字符串
  • getServletConfig 返回有Servlet容器穿个init方法的ServletConfig

3.注意线程安全：

Servlet实例会被一个应用程序中的所用用户共享，因此不建议使用类级变量，除非他们是只读的，或者是java.util.concurrent.atomic包的成员

编写基础的Servlet应用程序

1.标注

WebServlet标注了用来说明一个Servlet。name属性可选，如有通常用Servlet类的而名称。urlPatterns属性可选，用来说明调用Servlet的路径。

2.transient不可被序列化

3.throw和throws：throw明确表示这里会抛出何种类型的异常，thows表示这里可能跑出的异常。

4.如果想通过Servlet内部使用ServletConfig，只需要将被传入的ServletConfig赋给一个类变量

应用程序目录结构

  • classes Servlet类及其它Java类不许放在这里面。类一下的目录反映了类包的结构
  • lib Servlet应用此程序所需的JAR文件存放目录
  • 其它类似JSP页面、HTML页面、图片文件等，放置在应用程序目录下对应的文件夹中

放置在应用程序目录下的任何资源，只要用户输入资源URL，都可以直接访问到。如果想让某一个资源可以被Servlet访问，但不可以被用户访问，须将文件放入WEB-INF目录下。

ServletRequest

1.对于每一个HTTP请求，Servlet容器都会创建要给ServletRequest实例，并将它传给Servlet的Service方法。ServletRequest封装了关于这个请求的信息

2.ServletRequest接口中的方法：

  • public int getContentLength() 返回请求主题的字节数。如果不知道字节长度，返回-1
  • public java.lang.String getContentType() 返回请求主题的MIME类型，如果不知道类型，返回null
  • public java.lang.String getParameter(java.lang.String name) 返回指定请求参数的值
  • public java.lang.String getProtocol() 返回这个HTTP请求的协议名称和版本

3.getParameter是ServletRequest中最常用的方法。该方法通常用于返回HTML表单域的值

4.除了getParameter外，还可以使用getParameterNames、getParameterMap和getParameterValues获取表单域名、值以及查询字符串。

ServletResponse

1.javax.servlet.ServletResponse接口：表示一个Servlet响应。在调用Servert的Service方法前，Servelet容器首先创建一个ServletResponse，并将他作为第二个参数传给Servlet方法。

2.getWriter方法：定义在ServletResponse中，返回要给可以向客户端能发送文本的java.io.PrintWriter。默认情况系，PrintWriter对象使用ISO-8859-1编码

3.getOutputStream方法： 同样可以想浏览器发送输出，发送的是二进制数据

4.setContentType方法： 在发送任何THML标签前，需调用setContentType方法，设置响应的内容类型。在没有内容类型的情况下，大多数浏览器会默认将响应渲染成HTML

ServletConfig

当容器初始化Servlet时，Servlet容器会给Servlet的init方法传入一个ServletConfig。ServletConfig封装通过@WebServlet或者部署描述符（web.xml）传给Servlet的配置信息。这样传入的每一条信息就叫一个初始参数。一个初始参数有key和value两个元件。

1.getInitParameter方法：为了从Servlet内部获取到草书的值，咬字啊Servlet容器传给Servlet的init方法的ServletConfig中调用getInitParameter方法。getInitParameter的方法签名如下：

java.lang.String getInitParmeter(java.lang.String name)

2.getInitParameterNames方法：getInitParameterNames方法返回所有初始参数名称的一个Enumeration（枚举类）java.util.Enumeration<java.lang.String> getInitParameterNames()

3.getServletContext方法： 从Servlet内部获取ServletCongtext

GenericServlet

1.GenericServlet抽象类：

GenericServlet实现了Servlet和ServletConfig接口，并完成了一下任务：

  • 将init方法中的ServletConfig赋给一个类级变量，以便通过调用getServletConfig获取
  • 为Servlet接口中的所用方法提供默认的实现。
  • 提供方法，包围ServletConfig中的方法

2.GenericServlet可通过覆盖没有参数的init方法来编写初始化代码，ServletConfi则仍然有GenericServlet实例保存

Http Servlets

1.HttpServlet:

HttpServlet类继承了javax.servlet.GenericServlet类。使用HttpServlet时，还要借助HttpServletRquest和HttpServletResponse。

HttpServlet的两个特性：

  • 不用覆盖service方法，而是覆盖doGet、doPost等方法
  • 使用HttpServletRquest和HttpServletResponse

2.HttpServletRequest:

相对于ServletRequest新增的几个方法：

  • java.lang.String getContextPath() 返回表示请求上下文的请求URI部分
  • Cookie() getCookie() 返回一个Cookie对象数组
  • java.lang.String getMethod() 返回生成这个请求的HTTP方法名称
  • java.lang.String getHeader(java.lang.String name) 返回指定HTTP标题的值
  • java.lang.String getQueryString() 返回请求URL中的查询字符串
  • HttpSession getSession() 返回与这个请求相关的会话对象。如果没有则创建一个新的会话对象

HttpServeltResponse:

  • void addCookie(Cookie cookie) 给这个响应对象添加一个cookie
  • void addHeader(java.lang.String name,java.lang.String value) 给这个响应对象添加一个header
  • void sendRedirect(java.lang.String location) 发送一条响应码，将浏览器跳转到指定位置

使用部署描述符

1.部署描述符可以使用@WebServlet中没有对等元素的元素

2.修改配置值时，不需要重新编译Servlet类

3.部署描述符允许覆盖在Servlet标注中定义的值，及Servlet上的WebServlet标注如果同时也在部署描述符中进行声明，那么他将不起作用。
