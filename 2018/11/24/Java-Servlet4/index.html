Java-Servlet学习笔记（四）--JavaServer Page(JSP)

字数统计: 2,192字   |   阅读时长: 8分
  • Java
  • Servlet

前言

这部分记录一下Java web中特别重要的JSP

JSP概述

1.Servlet的两个缺点：

  • 写在Servlet中的所有HTML标签必须包含在Java字符串中
  • 所有的文本和HTML标记是硬编码，导致即使是表现层的微小变化也需要重新编译。

2.JSP的编写可以使用标准语法或XML语法，XML语法编写的JSP页面被称为JSP文档

3.当一个JSP页面第一次被请求时，Servlet/JSP容器进行的主要工作：

  • 转换JSP页面到JSP页面实现类，该实现类是一个实现javax.servlet.jsp.JspPage接口或子接口javax.servlet.jsp.HttpJspPage的Java类。JspPage是javax.servlet.Servlet的子接口，这使得每一个JSP页面都是要给Servlet。
  • 转后成功后，Servlet/JSP容器随后编译该类，并装载和实例化该类，像其它正常的Servlet一样执行生命周期操作

4.对同一个JSP页面的后续请求，Servlet/JSP容器会先检查JSP页面是否被修改过。如果被修改则该JSP页面会被重新翻译、编译并执行。如果不是，则执行已经在内存中的JSP Servlet。由此会导致一个JSP页面在第一次调用的实际花费总比后面的花费多。

5.解决方法：

  • 配置应用程序，是所有的JSP页面在应用程序启动时被调用，而不是在第一次请求时调用。
  • 预编译JSP页面，并将其部署为Servlet。

6.JSP自带API包含的四个包：

  • javax.servlet.jsp 包含用于Servl/JSP容器将JSP页面翻译为Servlet的核心类和接口。其中最为重要的是JspPage和HttpJspPage接口。所有的JSP页面实现类必须实现这两个接口。
  • javax.servlet.jsp.tagext 包括用于开发自定义标签的类型。
  • javax.el 提供统一表达式语言的API
  • javax.servlet.jsp.el 提供一组必须由Servlet/JSP容器支持，以便在JSP页面中使用表达式语言的类。

7.JSP页面包含语法元素和模板元素：

  • 语法元素 一些具有特殊意义的JSP转化符
  • 模板元素 出去语法元素外的一切都是模板元素，例如JSP页面中的HTML标记和文字都是模板元素，模板元素会原样发送给浏览器

8.在应用程序目录中的每一个JSP页面都可以直接在浏览器中输入路径页面访问

9.Java代码可以出现在JSP页面中的任何位置，并通过”<%”和”%>”包括起来，其次可以使用page指令的import属性导入在JSP页面中使用的Java类型，如果没有导入则需要在代码中写Java类的全路径名称。

隐式对象

对象 类型
request javax.servlet.http.HttpServletRequest
response javax.servlet.http.HttpServletResponse
out javax.servlet.jsp.JspWriter
session javax.servlet.http.HttpSession
application javax.servlet.ServletContext
config javax.servlet.ServletConfig
pageContext javax.servlet.jsp.PageContext
page javax.servlet.jsp.HttpJspPage
exception java.lang.Throwable

以request为例，该隐式对象代表Servlet/JSP容器传递给Servlet服务方法的HttpServletRequest对象。可以将request理解为一个指向HttpServletRequest对象的引用变量。

1.PageContext用于获取和设置属性的方法：setAttribute和getAttribute方法。属性值可以被设置在四个范围之一：页面（PAGE_SCOPE）、请求（REQUEST_SCOPE）、会话（SESSION_SCOPE）和应用程序（APPLICATION_SCOPE）

2.隐式对象out引用了一个javax.servlet.jsp.JspWriter对象，这类似于在调用HttpServletResponse的getWriter方法得到java.ioPrintWriter。可以通过调用它的print方法将消息发送到浏览器

3.默认情况下，JSP编译器会将JSP页面的内容类型设为text/html

指令

可以使用page指令来控制JSP转换器转换当前JSP页面的某些方面

1.page指令的属性列表：

  • import 定义一个或多个本页面中将被导入和使用的Java类型包，可以使用通配符*导入整个包，两个类型之间用逗号分隔导入多个类型，JSP默认导入如下包：java.lang、javax.servlet、javax.servlet.http、javax.servlet.jsp
  • session 值为true，本页面加入会话管理。默认值为True。访问该页面时，若当前不存在javax.servlet.http.HttpSession实例，则会创建一个
  • buffer 以KB为单位，定义隐式对象out的缓存大小。该值可以为none，这意味着没有缓冲，所有数据直接写入PrintWriter
  • info 返回调用容器生成的Servlet类的getServletInfo方法的结果
  • errorPage 定义当出错时用来处理错误的界面
  • isErrorPage 表示本页是一个错误处理页面
  • contentType 定义本页面隐式对象response的内容类型，默认是text/html
  • pageEncoding 定义本页面的字符编码，默认是ISO-8859-1
  • language 定义本页面的脚本语言类型，默认是Java，这是在JSP2.2中唯一的合法值
  • extends 定义JSP实现类要继承的父类

2.大部分page之类可以出现在页面的任何位置，但当page指令包含contentType或pageEncoding属性时，器必须出现在Java代码发送任何内容之前。这是因为内容类型和字符编码必须在发送任何内容前设定

3.include指令：可以使用include指令将其它文件中的内容包含到当前JSP页面

URL为被包含文件的相对路径，若URL以斜杠（/）开头，则该URL为文件在服务器上的绝对路径

3.JSRF文件：以JSPF为扩展名的文件代表JSP fragement。虽然JSP fragement现在被称为JSP segement，但为了保证一致性，任使用JSPF为后缀名

4。include指令也可以包含静态HTML文件

脚本元素

1.表达式：每个表达式都会被JSP容器执行，并使用隐式对象out的打印方法输出结果。表达式以”<%=”开始，并以”%>”结束

表达式无需分号结尾

2.声明：可以声明在JSP页面中使用的变量和方法。声明以”<%!”开始，并以”%>”结尾。

3.可以使用声明来重写JSP页面，，实现类的init和destroy方法。通过声明jspInit方法来重写init方法。声明jspDestroy方法来重写destroy方法，两种方法说明如下：

  • jspInit 这种发发类似与javax.servlet.Servlet的init方法。JSP页面在初始化时调用jdpInit，jspInit没有参数，可以通过隐式对象config访问ServletConfig对象
  • jspDestroy 这种方法类似于Servlet的destroy方法，在JSP页面被销毁时调用

4.禁用脚本元素：

随着JSP2.0对表达式语言的加强，推荐使用的方法是在JSP页面中用EL访问服务器端对象且不写Java代码。

从JSP2.0起，可以通过在部署描述符中的\定义一个scripting-invalid元素来禁用脚本元素

动作

动作是第三种类型的语法元素，它们被转换成Java代码来执行

1.useBean

useBean将创建一个关联Java对象的脚本变量。这是早期分离表现层和业务逻辑的表现，语法如下

2.setProperty和getProperty

setProperty动作可对一个Java对象设置属性，而getProperty则会输出Java对象的一个属性

3.include

include动作用来动态地引入一个资源。可以引入另一个JSP页面，也可以引入一个Servlet或一个静态的HTML页面。

4.include动作和include指令的区别：

include指令，资源的引入发生在页面转换时，及当JSP容器将页面转换为生成的Servlet时。而对于include动作，资源引入发生在请求页面时。因此，使用include动作是可以传递参数的，而include指令不支持。

除了在Java代码中使用try语句外，还可以指定一个特殊页面。当引用页面遇到未捕获的异常时，将显示这个特殊的界面

使用page指令的isErrorPage属性（属性值必须为true）来标识一个JSP页面是错误处理页面

需要防止正常的页面使用page指令的errorPage属性来导向错误处理界面
