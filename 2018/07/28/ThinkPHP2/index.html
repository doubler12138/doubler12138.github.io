前言

这一部分学习PHP开发框架ThinkPHP的路由部分

URL的三种模式

动态URL：也指动态页面。优点：有很强的交互性；缺点：安全隐患大，每次访问都需要经过服务器的编译执行，对服务器有一定的负载压力

静态URL：也指静态页面。优点：不做任何编译操作，访问速度快、跨平台、跨服务器；缺点：文件直接村房子啊服务器磁盘上，服务器磁盘空间会占用过多

伪静态URL：本质还是动态页面。优点：解决了前两者的问题；缺点：到底发送什么内容由Web服务器来判定，访问量大时容易导致网站崩溃

ThinkPHP的路由

路由模式

ThinkPHP支持一下四种模式：

  • 普通模式
  • pathinfo模式
  • rewrite模式
  • 兼容模式

普通模式和兼容模式归类到动态URL，pathinfo模式和rewrite模式归类到伪静态模式。想要以rewrite模式访问页面需要写重定向规则。

路由定义的一般规则：“路由表达式” => “路由地址和参数” 或者array(“路由表达式”，“路由地址”，“传出参数”)。

路由表达式是指以何种规则匹配浏览器中的地址，如果匹配成功，系统将在处理请求的同时把传入参数传给指定的动作。ThinkPHP的路由规则采取顺序遍历方式进行，只要成功匹配一条匹配的路由，则终止继续匹配

路由表达式

规则路由：通过“:”进行参数匹配，如果匹配陈工，则将该位置的参数传给指定动作

正则路由：通过正则匹配进行路由表达式的配置

静态路由：静态路由定义中不含任何动作参数，也不需要遍历路由规则。为了区分前两种路由，静态路由采用URL_MAP_RULES进行定义。
